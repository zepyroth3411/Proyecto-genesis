Documentación del Proyecto

1.Introduccion

Proyecto nace como una práctica universitaria...

1.Introducción del proyecto

Este proyecto nace como una práctica universitaria con el objetivo de demostrar cómo es posible controlar dispositivos físicos, como una garra robótica, de manera remota y eficiente mediante tecnología moderna. Aunque sencillo en su concepción, el prototipo introduce conceptos fundamentales de electrónica y programación, y ofrece una experiencia completa en desarrollo full-stack al integrar hardware y software.

El ejemplo de la garra robótica puede escalarse hacia aplicaciones más complejas, como el control de PLCs, sistemas embebidos, brazos robóticos industriales o simulaciones de control remoto en sectores como la industria y la exploración remota. Además, en entornos educativos, sirve como introducción práctica a la programación y la electrónica básica.

En resumen, este proyecto no solo cumple objetivos académicos, sino que también demuestra el potencial de herramientas tecnológicas accesibles para resolver problemas reales y explorar soluciones innovadoras.

2.Arquitectura

Conecta al usuario, el servidor y el hardware...

2.Arquitectura del sistema

La arquitectura del sistema conecta al usuario, el servidor y el hardware para lograr el control eficiente de la garra robótica. Esta integración entre software y hardware está diseñada de manera modular, permitiendo escalabilidad y adaptabilidad a diferentes aplicaciones.

Componentes del sistema

1.Cliente (Frontend) El cliente se representa mediante una interfaz web intuitiva y accesible desde cualquier navegador. A través de esta interfaz, el usuario puede enviar comandos específicos para interactuar con la garra robótica, como abrir, cerrar, o moverla en distintas direcciones. Los comandos generados por el usuario son enviados al servidor utilizando solicitudes HTTP o WebSockets, dependiendo de los requerimientos de comunicación en tiempo real.


2.Servidor (Backend)El servidor se encuentra alojado en un entorno que procesa y valida las solicitudes enviadas desde el cliente. Este componente traduce los comandos en un formato que el NodeMCU pueda interpretar y los transmite al hardware a través de una red local (Wi-Fi). Adicionalmente, el servidor puede realizar funciones como:
  • Autenticación de usuarios.
  • Registro de comandos para análisis posterior.
  • Notificación de errores en caso de fallas en la ejecución.

    • 3.Hardware (NodeMCU y garra robótica) El NodeMCU es el microcontrolador central del sistema, que se conecta al servidor mediante Wi-Fi. Este recibe comandos, los procesa y controla los motores o actuadores de la garra robótica a través de sus pines GPIO. En caso de incluir sensores, el NodeMCU también puede enviar retroalimentación al servidor sobre el estado del sistema, como la posición actual de la garra o posibles errores.

    3.Codigo Fuente

    Controlar una garra robótica a través de una conexión Wi-Fi...

    3.Código Fuente: Control de la Garra Robótica

    El programa está diseñado para controlar una garra robótica a través de una conexión Wi-Fi, utilizando un microcontrolador ESP8266 o ESP32. A continuación, se describen los principales componentes del código y su funcionalidad.

    1.Bibliotecas Utilizadas
  • ESP8266WiFi.h / wifi.h

    • Permite la conexión del microcontrolador a una red Wi-Fi. En caso de usar un ESP32, se debe cambiar la biblioteca a WiFi.h.

  • 2.ESP8266WebServer.h / WebServer.h

    • Implementa un servidor web que procesa las solicitudes HTTP enviadas por el cliente. Cambia a WebServer.h si utilizas un ESP32.

  • 3.Servo.h

    • Biblioteca para controlar los servos de la garra robótica. Proporciona funciones para manejar ángulos y movimientos.

    3.1.Codigo fuente

    Primeros pasos...

    3.1.Codigo fuente: Configuración de Red Wi-Fi

    El código utiliza las variables ssid y password para conectarse a una red Wi-Fi. Estas deben ser configuradas con el nombre de la red y la contraseña correspondientes.

  • const char* ssid = "TuSSID"; // Cambia por tu red WiFi
  • const char* password = "TuPassword"; // Cambia por tu contraseña WiFi

    • 2.Servos y Posiciones Iniciales

    Se emplean cuatro servos, cada uno asignado a una parte de la garra robótica:

  • xServo:Movimiento en el eje X.
  • yServo:Movimiento en el eje Y.
  • zServo:Movimiento en el eje Z.
  • clawServo:Controla la apertura y cierre de la garra.

    • 3.Cada servo se inicializa en una posición inicial de 90 grados.

  • int xPos = 90;
  • int yPos = 90;
  • int zPos = 90;
  • int clawPos = 90;

    • En el método setup, los servos se conectan a pines específicos del microcontrolador:

  • xServo.attach(11);
  • y.attach(10);
  • zServo.attach(9);
  • clawnServo.attach(6);

    • 3.2.Codigo fuente

    Primeros pasos...

    3.2.Codigo fuente: Configuración del servidor web

    El servidor web se inicializa en el puerto 80 y se configura para manejar solicitudes HTTP. Una de las rutas principales es /move, la cual se utiliza para recibir comandos de movimiento para los servos.

  • server.on("/move", handleMove); // Ruta para mover el brazo
  • server.begin();

    • Cuando el servidor recibe una solicitud en la ruta /move, se llama a la función handleMove, encargada de interpretar los parámetros enviados por el cliente.

    4.Función handleMove

    La función procesa los parámetros enviados en la URL (por ejemplo, ?x=45&y=90&z=135&claw=60) y ajusta los ángulos de los servos utilizando la biblioteca Servo.

  • if (server.hasArg("x")) xPos = constrain(server.arg("x").toInt(), 0, 180);
  • if (server.hasArg("y")) yPos = constrain(server.arg("y").toInt(), 0, 180);
  • if (server.hasArg("z")) zPos = constrain(server.arg("z").toInt(), 0, 180);
  • if (server.hasArg("claw")) clawPos = constrain(server.arg("claw").toInt(), 0, 180);

    • Los valores son validados para garantizar que se encuentren en el rango de 0 a 180 grados. Después, los servos se mueven a las posiciones correspondientes:

  • xServo.write(xPos);
  • yServo.write(yPos);
  • zServo.write(zPos);
  • clawServo.write(clawPos);

    • Finalmente, se envía una respuesta al cliente indicando que el movimiento fue ejecutado:

  • server.send(200, "text/plain", "Movimiento ejecutado");

    • 4.Flujo

    4. Flujo del Programa

    1.Conexión Wi-Fi:

    El microcontrolador intenta conectarse a la red especificada. Una vez conectado, muestra la dirección IP asignada en el monitor serial.

    2.Inicio del Servidor Web:

    El servidor queda a la espera de solicitudes en el puerto 80.

    3.Manejo de Solicitudes:

  • El cliente envía una solicitud HTTP a la ruta /move con los parámetros deseados.
  • El servidor interpreta los parámetros y ajusta los servos en consecuencia.

    • 4.Movimiento de la Garra:

    Los servos se mueven a las posiciones indicadas, y se envía una confirmación al cliente.
